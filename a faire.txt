4. Wizard de Lancement Formation — Architecture (Frontend)
C'est la fonctionnalité centrale. Voici les 5 étapes du wizard React :

text
Étape 1 — Infos Formation
  ├── Sélectionner une formation existante (autocomplete)
  └── Ou créer une nouvelle (titre, description, tarif, dates, masse horaire)

Étape 2 — Formateurs
  ├── Rechercher formateurs existants → ajouter
  ├── Créer un nouveau formateur (mini-form inline)
  └── Pour chaque formateur : spécifier mode honoraire
       (horaire | par étudiant | forfait | mensualité) + montant

Étape 3 — Étudiants
  ├── Rechercher étudiants existants → inscrire
  ├── Créer nouveau compte étudiant (inline)
  └── Pour chaque étudiant : taux de réduction (défaut 0%)

Étape 4 — Paiements Étudiants
  └── Pour chaque étudiant inscrit :
       mode paiement (total | par mois | par tranche | à volonté)
       + montant / nombre de mois / montant par tranche

Étape 5 — Récapitulatif & Confirmation
  ├── Tableau récap : formation, formateurs, étudiants, montants
  └── Bouton "Lancer la formation"
Le WizardDraft est déjà dans le modèle pour sauvegarder l'état et permettre de reprendre. Utilise le champ data_json pour stocker l'état de chaque étape.

5. Module Pack — Ce qui manque
Le modèle Pack a discount_rate mais il manque :

python
# Ajouter dans Pack :
description = Column(Text, nullable=True)
created_at = Column(DateTime(timezone=True), server_default=func.now())
Côté logique : le prix final du pack = somme(formation.price) * (1 - pack.discount_rate). Cette formule doit être calculée dans le service au moment de l'inscription, pas stockée comme champ fixe (car les prix des formations peuvent changer).

Par Formateur :
